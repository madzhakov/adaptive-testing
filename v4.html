Project Concept: AI-Driven Adaptive Testing

The core concept is to use a powerful Machine Learning model, XGBoost, to create an adaptive test flow. Instead of testing every parameter on every die, we will train a model on data from an early test stage (PR1) to predict the outcome of a long, expensive test in a later stage (PR2). If the model is confident a part will pass, we can safely skip that test; if it flags a part as high-risk, we run the test. This selective approach, known as adaptive testing, saves significant time and cost while maintaining quality.

Data Files: All necessary data files for your assigned project can be found here: Project Data Folder

Phase 1 Task Checklist

  1. Ingest and Normalize: Read PR1 and PR2, including header rows for limits and units. Standardize keys device_id, lot_id, wafer, x, y.
  2. Merge PR1 to PR2: Inner join on the keys. Check counts before and after the merge to confirm you did not lose devices. Save merged_dataset.csv.
  3. Pareto of Time: Use test_times_yield_by_cpp.csv to create a time Pareto by .cpp. Select one PR2 target and give a short reason based on time share and risk.
  4. Define the PR2 Target: Choose the long or expensive PR2 test (or a small group). Map PASS and FAIL to 1 and 0. Note the overall pass rate.
  5. Target Visualization:
    • Plot histograms of the PR2 target tests, which are included in the .cpp
    • Plot a bar chart of number of fails per test within the .cpp
  6. Quick Sanity: Features must be PR1 only, the merge coverage looks reasonable, and your PASS and FAIL mapping is written in the notebook.

Phase 2: ML Model Training and Analysis

With a clean dataset and a clear PR2 target, this phase is focused on building, evaluating, and refining our predictive model.

Your Goals for this Phase

  1. Train an XGBoost model using PR1 features to predict the PR2 target.
  2. Use grouped validation by lot to estimate real performance.
  3. Select a decision threshold that meets the escape rate requirement. Report skip rate and time saving.
  4. Review feature importance and write one or two engineering insights.
  5. Optional: investigate any test escapes to understand why they were missed.

Key Concepts for Model Analysis

Confusion Matrix

A summary of correct predictions and errors. False negatives are test escapes. False positives are yield loss.

Feature Importance

Which PR1 measurements were most useful for predictions.

Threshold by Constraint

Choose the lowest probability threshold that keeps escapes under the ppm target. Report the resulting skip rate and time saving.

Grouped Validation

Validate by lot to simulate new production. Optionally check by wafer.
